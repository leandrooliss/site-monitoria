Sobre a Monitoria

A monitoria de Algoritmos e Lógica de Programação foi criada para apoiar os alunos da FATEC Rubens Lara no entendimento dos conceitos fundamentais da programação. Este espaço reúne materiais didáticos, exemplos práticos, listas de exercícios e links úteis para facilitar o aprendizado e a prática dos conteúdos abordados em sala de aula.

Aqui você encontrará explicações sobre algoritmos, lógica de programação, estruturas de controle, ordenação, estruturas de dados e muito mais. O objetivo é tornar o estudo mais acessível, interativo e colaborativo, com suporte da monitoria e colegas.

Seja bem-vindo! Explore os conteúdos, participe dos grupos e aproveite os recursos disponíveis para evoluir na sua jornada como programador.

Conteúdo da Disciplina

Visão Geral

Esta disciplina introduz os conceitos fundamentais de algoritmos e lógica de programação, preparando os alunos para desenvolver soluções computacionais estruturadas. Os principais tópicos abordados incluem:


  • Algoritmos: Definição, características, exemplos práticos e formas de representação como fluxogramas, pseudocódigo e descrição narrativa.
  • Lógica de Programação: Conceitos de sequência lógica, operadores, atribuição e estruturação de pensamento para resolução de problemas.
  • Estruturas de Controle: Introdução às instruções condicionais (SE, SENÃO) e de repetição (ENQUANTO, PARA), que permitem controlar o fluxo dos programas.

Use o menu lateral para acessar os conteúdos detalhados de cada tópico, com explicações, exemplos e exercícios.

Algoritmos

Um algoritmo é uma sequência finita de passos bem definidos que descrevem como resolver um problema ou realizar uma tarefa. Ele funciona como uma receita: se seguido corretamente, leva a um resultado esperado.


Características de um bom algoritmo:


  • Precisão: cada instrução deve ser clara e objetiva.
  • Não ambiguidade: os passos não podem gerar interpretações diferentes.
  • Finitude: deve terminar após um número limitado de etapas.
  • Eficiência: deve usar bem os recursos disponíveis (tempo, memória).
  • Generalidade: deve resolver um problema genérico, não apenas um caso específico.

Exemplos de algoritmos:


  • Receita de bolo: lista de ingredientes e passos para preparar.
  • Instruções para montar um móvel: sequência de ações para chegar ao resultado final.
  • Fluxo de login em um sistema: verificar usuário, senha e liberar acesso.

Formas de representar algoritmos:


  • Descrição narrativa: escrita em linguagem natural.
  • Fluxograma: uso de símbolos gráficos para representar o fluxo lógico.
  • Pseudocódigo: escrita estruturada com comandos semelhantes aos de programação.

Os algoritmos são a base da programação. Antes de escrever um código, é essencial entender e planejar o algoritmo que resolverá o problema.

Lógica de Programação

A lógica de programação é a base para construir algoritmos e desenvolver programas. Ela envolve o raciocínio necessário para organizar instruções de forma que resolvam um problema de maneira eficiente e estruturada.


Conceitos fundamentais:


  • Sequência lógica: conjunto ordenado de passos que levam à solução de um problema.
  • Linguagem de programação: conjunto de regras sintáticas e semânticas usadas para escrever programas que o computador possa interpretar.
  • Operadores: símbolos que realizam operações matemáticas, relacionais ou lógicas (ex.: +, >, E, OU).
  • Atribuição: ação de armazenar um valor em uma variável (ex.: a = 100).

Etapas para desenvolver um programa:


  1. Análise: entender o problema e identificar os dados de entrada, processamento e saída.
  2. Algoritmo: descrever a solução usando narrativa, fluxograma ou pseudocódigo.
  3. Codificação: transformar o algoritmo em código usando uma linguagem de programação.

Aprender lógica de programação é essencial para dominar qualquer linguagem. É como aprender a pensar como um computador: de forma clara, precisa e estruturada.

Estrutura de Controle

As estruturas de controle são responsáveis por definir o caminho que o programa deve seguir durante sua execução. Elas permitem tomar decisões e repetir ações com base em condições específicas.


Tipos principais:


  • Seleção (Condicional): executa diferentes blocos de código dependendo de uma condição. Exemplo: SE...SENÃO.
  • Repetição (Laços): executa um bloco de código várias vezes enquanto uma condição for verdadeira. Exemplos: ENQUANTO, PARA.

Exemplo de estrutura condicional:

            Verificação de idade:
            SE (idade ≥ 18) ENTÃO
                ESCREVA "Maior de idade"
            SENÃO
                ESCREVA "Menor de idade"
            FIM SE

Exemplo de estrutura de repetição:

            Contagem de 1 a 5:
            PARA i = 1 ATÉ 5 FAÇA
                ESCREVA i
            FIM PARA

Essas estruturas são fundamentais para criar programas dinâmicos e inteligentes, que se adaptam a diferentes situações e dados de entrada.
